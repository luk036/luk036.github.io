189  // completed is set if a Group is iterated through
190  // completely. It is checked in the destructor, and
191  // if the Group has not been completed, the destructor
192  // exhausts it. This ensures that the next Group starts
193  // at the correct position when the user short-circuits
194  // iteration over a Group.

196  // attribute to true, so its destructor doesn't do anything
197  // when called.
