16 /*! Base class for directed graphs.
17 
18  A DiGraphS stores nodes and edges with optional data, or attributes.
19 
20  DiGraphSs hold directed edges. Self loops are allowed but multiple
21  (parallel) edges are not.
22 
23  Nodes can be arbitrary (hashable) C++ objects with optional
24  key/value attributes. By convention `None` is not used as a node.
25 
26  Edges are represented as links between nodes with optional
27  key/value attributes.
28 
29  Parameters
30  ----------
31  node_container : input graph (optional, default: None)
32  Data to initialize graph. If None (default) an empty
33  graph is created. The data can be any format that is supported
34  by the to_networkx_graph() function, currently including edge list,
35  dict of dicts, dict of lists, NetworkX graph, NumPy matrix
36  or 2d ndarray, SciPy sparse matrix, or PyGraphviz graph.
37 
38  See Also
39  --------
40  Graph
41  DiGraph
42  MultiGraph
43  MultiDiGraph
44  OrderedGraph
45 
46  Examples
47  --------
48  Create an empty graph structure (a "null graph") with 5 nodes and
49  no edges.
50 
51  >>> auto v = std::vector{3, 4, 2, 8};
52  >>> auto G = xn::DiGraphS(v);
53 
54  >>> auto va = py::dict{{3, 0.1}, {4, 0.5}, {2, 0.2}};
55  >>> auto G = xn::DiGraphS(va);
56 
57  >>> auto r = py::range(100);
58  >>> auto G = xn::DiGraphS(r);
59 
60  G can be grown in several ways.
61 
62  **Nodes:**
63 
64  Add one node at a time:
65 
66  >>> G.add_node(1)
67 
68  Add the nodes from any container (a list, dict, set or
69  even the lines from a file or the nodes from another graph).
70 
71  >>> G.add_nodes_from([2, 3])
72  >>> G.add_nodes_from(range(100, 110))
73  >>> H = xn::path_graph(10)
74  >>> G.add_nodes_from(H)
75 
76  In addition to strings and integers any hashable C++ object
77  (except None) can represent a node, e.g. a customized node object,
78  or even another DiGraphS.
79 
80  >>> G.add_node(H)
81 
82  **Edges:**
83 
84  G can also be grown by adding edges.
85 
86  Add one edge,
87 
88  >>> G.add_edge(1, 2);
89 
90  a list of edges,
91 
92  >>> G.add_edges_from([(1, 2), (1, 3)]);
93 
94  or a collection of edges,
95 
96  >>> G.add_edges_from(H.edges());
97 
98  If some edges connect nodes not yet in the graph, the nodes
99  are added automatically. There are no errors when adding
100  nodes or edges that already exist.
101 
102  **Attributes:**
103 
104  Each graph can hold key/value attribute pairs
105  in an associated attribute dictionary (the keys must be hashable).
106  By default these are empty, but can be added or changed using
107  direct manipulation of the attribute
108  dictionaries named graph, node and edge respectively.
109 
110  >>> G.graph["day"] = std::any("Friday");
111  {'day': 'Friday'}
112 
113  **Subclasses (Advanced):**
114 
115  The DiGraphS class uses a container-of-container-of-container data structure.
116  The outer dict (node_dict) holds adjacency information keyed by node.
117  The next dict (adjlist_dict) represents the adjacency information and holds
118  edge data keyed by neighbor. The inner dict (edge_attr_dict) represents
119  the edge data and holds edge attribute values keyed by attribute names.
120 
121  Each of these three dicts can be replaced in a subclass by a user defined
122  dict-like object. In general, the dict-like features should be
123  maintained but extra features can be added. To replace one of the
124  dicts create a new graph class by changing the class(!) variable
125  holding the factory for that dict-like structure. The variable names are
126  node_dict_factory, node_attr_dict_factory, adjlist_inner_dict_factory,
127  adjlist_outer_dict_factory, edge_attr_dict_factory and
128  graph_attr_dict_factory.
129 
130  node_dict_factory : function, (default: dict)
131  Factory function to be used to create the dict containing node
132  attributes, keyed by node id.
133  It should require no arguments and return a dict-like object
134 
135  node_attr_dict_factory: function, (default: dict)
136  Factory function to be used to create the node attribute
137  dict which holds attribute values keyed by attribute name.
138  It should require no arguments and return a dict-like object
139 
140  adjlist_outer_dict_factory : function, (default: dict)
141  Factory function to be used to create the outer-most dict
142  in the data structure that holds adjacency info keyed by node.
143  It should require no arguments and return a dict-like object.
144 
145  adjlist_inner_dict_factory : function, (default: dict)
146  Factory function to be used to create the adjacency list
147  dict which holds edge data keyed by neighbor.
148  It should require no arguments and return a dict-like object
149 
150  edge_attr_dict_factory : function, (default: dict)
151  Factory function to be used to create the edge attribute
152  dict which holds attribute values keyed by attribute name.
153  It should require no arguments and return a dict-like object.
154 
155  graph_attr_dict_factory : function, (default: dict)
156  Factory function to be used to create the graph attribute
157  dict which holds attribute values keyed by attribute name.
158  It should require no arguments and return a dict-like object.
159 
160  Typically, if your extension doesn't impact the data structure all
161  methods will inherit without issue except: `to_directed/to_undirected`.
162  By default these methods create a DiGraph/DiGraphS class and you probably
163  want them to create your extension of a DiGraph/DiGraphS. To facilitate
164  this we define two class variables that you can set in your subclass.
165 
166  to_directed_class : callable, (default: DiGraph or MultiDiGraph)
167  Class to create a new graph structure in the `to_directed` method.
168  If `None`, a NetworkX class (DiGraph or MultiDiGraph) is used.
169 
170  to_undirected_class : callable, (default: DiGraphS or MultiGraph)
171  Class to create a new graph structure in the `to_undirected` method.
172  If `None`, a NetworkX class (DiGraphS or MultiGraph) is used.
173 
174  Examples
175  --------
176 
177  Create a low memory graph class that effectively disallows edge
178  attributes by using a single attribute dict for all edges.
179  This reduces the memory used, but you lose edge attributes.
180 
181  >>> class ThinGraph(xn::DiGraphS):
182  ... all_edge_dict = {'weight': 1}
183  ... def single_edge_dict(self):
184  ... return self.all_edge_dict
185  ... edge_attr_dict_factory = single_edge_dict
186  >>> G = ThinGraph()
187  >>> G.add_edge(2, 1)
188  >>> G[2][1]
189  {'weight': 1}
190  >>> G.add_edge(2, 2)
191  >>> G[2][1] is G[2][2]
192  True
193 
194  Please see :mod:`~networkx.classes.ordered` for more examples of
195  creating graph subclasses by overwriting the base class `dict` with
196  a dictionary-like object.
197 */

234  /// @property
235  /*! DiGraphS adjacency object holding the neighbors of each node.
236 
237  This object is a read-only dict-like structure with node keys
238  and neighbor-dict values. The neighbor-dict is keyed by neighbor
239  to the edge-data-dict. So `G.adj[3][2]['color'] = 'blue'` sets
240  the color of the edge `(3, 2)` to `"blue"`.
241

243  `for nbr, datadict in G.adj[n].items():`.
244

246  So `for nbr, foovalue in G[node].data('foo', default=1):` works.
247 
248  For directed graphs, `G.adj` holds outgoing (successor) info.
249  */

254 
255  This object is a read-only dict-like structure with node keys
256  and neighbor-dict values. The neighbor-dict is keyed by neighbor
257  to the edge-data-dict. So `G.succ[3][2]['color'] = 'blue'` sets

397  If True, return edge attribute dict in 3-tuple (u, v, ddict).
398  If False, return 2-tuple (u, v).
