5 ## Used technologies
6 
7 Project is built on these technologies:
8 
9 * Git
10  - Source control
11 * CMake
12  - Used as build tool
13  - Doxygen generation
14  - CTest - for running tests
15  - CPack - for packaging the releases
16  + NSIS - windows installer
17  + RPM / DEB - linux packages
18  + compressed archives
19 * Doxygen
20  - For documentation and API reference generation
21  - Generates wiki from markdown files
22  - Custom modern style included
23  + to try modern style see doc/doxygen/Doxyfile.in line 3
24  + if you do not care feel free to delete `doc/doxygen/*.html` and `doc/doxygen/*.ccs`
25  - Graphviz - for creating UML diagrams with doxygen
26 * clang-format
27  - Configuration file in the root of project for easy formating
28  - Clang-format automatically uses nearest *.clang-format* file
29  - Target *format* will run format on all source files
30 * clang-tidy
31  - With target *tidy* you can run static code analysis
32 * cppcheck
33  - With target *cppcheck* you can run static code analysis
34 * [Catch](https://github.com/philsquared/Catch)
35  - Modern unit test framework
36  - Downloaded via externalproject from *external/CMakeLists.txt*
37 
38 
39 
40 There is extra support for these:
41 
42 * .ycm_extra_conf.py for ycmd support
43  - Smart autocompletion, goto def, etc.
44  - Works in Vim, Emacs, Sublime Text 3, Atom, Visual Studio Code
45  - Site: [github.com/Valloric/ycmd](https://github.com/Valloric/ycmd)
46 * Sublime text 3 project file
47  - with build system targets
