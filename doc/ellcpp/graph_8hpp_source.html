15 /*! Base class for undirected graphs.
16 
17  A Graph stores nodes and edges with optional data, or attributes.
18 
19  Graphs hold undirected edges. Self loops are allowed but multiple
20  (parallel) edges are not.
21 
22  Nodes can be arbitrary (hashable) C++ objects with optional
23  key/value attributes. By convention `None` is not used as a node.
24 
25  Edges are represented as links between nodes with optional
26  key/value attributes.
27 
28  Parameters
29  ----------
30  node_container : input graph (optional, default: None)
31  Data to initialize graph. If None (default) an empty
32  graph is created. The data can be any format that is supported
33  by the to_networkx_graph() function, currently including edge list,
34  dict of dicts, dict of lists, NetworkX graph, NumPy matrix
35  or 2d ndarray, SciPy sparse matrix, or PyGraphviz graph.
36 
37  See Also
38  --------
39  DiGraph
40  MultiGraph
41  MultiDiGraph
42  OrderedGraph
43 
44  Examples
45  --------
46  Create an empty graph structure (a "null graph") with 5 nodes and
47  no edges.
48 
49  >>> auto v = std::vector{3, 4, 2, 8};
50  >>> auto G = xn::Graph(v);
51 
52  >>> auto va = py::dict{{3, 0.1}, {4, 0.5}, {2, 0.2}};
53  >>> auto G = xn::Graph(va);
54 
55  >>> auto r = py::range(100);
56  >>> auto G = xn::Graph(r);
57 
58  G can be grown in several ways.
59 
60  **Nodes:**
61 
62  Add one node at a time:
63 
64  >>> G.add_node(1)
65 
66  Add the nodes from any container (a list, dict, set or
67  even the lines from a file or the nodes from another graph).
68 
69  >>> G.add_nodes_from([2, 3])
70  >>> G.add_nodes_from(range(100, 110))
71  >>> H = xn::path_graph(10)
72  >>> G.add_nodes_from(H)
73 
74  In addition to strings and integers any hashable C++ object
75  (except None) can represent a node, e.g. a customized node object,
76  or even another Graph.
77 
78  >>> G.add_node(H)
79 
80  **Edges:**
81 
82  G can also be grown by adding edges.
83 
84  Add one edge,
85 
86  >>> G.add_edge(1, 2);
87 
88  a list of edges,
89 
90  >>> G.add_edges_from([(1, 2), (1, 3)]);
91 
92  or a collection of edges,
93 
94  >>> G.add_edges_from(H.edges());
95 
96  If some edges connect nodes not yet in the graph, the nodes
97  are added automatically. There are no errors when adding
98  nodes or edges that already exist.
99 
100  **Attributes:**
101 
102  Each graph can hold key/value attribute pairs
103  in an associated attribute dictionary (the keys must be hashable).
104  By default these are empty, but can be added or changed using
105  direct manipulation of the attribute
106  dictionaries named graph, node and edge respectively.
107 
108  >>> G.graph["day"] = std::any("Friday");
109  {'day': 'Friday'}
110 
111  **Subclasses (Advanced):**
112 
113  The Graph class uses a container-of-container-of-container data structure.
114  The outer dict (node_dict) holds adjacency information keyed by node.
115  The next dict (adjlist_dict) represents the adjacency information and holds
116  edge data keyed by neighbor. The inner dict (edge_attr_dict) represents
117  the edge data and holds edge attribute values keyed by attribute names.
118 
119  Each of these three dicts can be replaced in a subclass by a user defined
120  dict-like object. In general, the dict-like features should be
121  maintained but extra features can be added. To replace one of the
122  dicts create a new graph class by changing the class(!) variable
123  holding the factory for that dict-like structure. The variable names are
124  node_dict_factory, node_attr_dict_factory, adjlist_inner_dict_factory,
125  adjlist_outer_dict_factory, edge_attr_dict_factory and
126  graph_attr_dict_factory.
127 
128  node_dict_factory : function, (default: dict)
129  Factory function to be used to create the dict containing node
130  attributes, keyed by node id.
131  It should require no arguments and return a dict-like object
132 
133  node_attr_dict_factory: function, (default: dict)
134  Factory function to be used to create the node attribute
135  dict which holds attribute values keyed by attribute name.
136  It should require no arguments and return a dict-like object
137 
138  adjlist_outer_dict_factory : function, (default: dict)
139  Factory function to be used to create the outer-most dict
140  in the data structure that holds adjacency info keyed by node.
141  It should require no arguments and return a dict-like object.
142 
143  adjlist_inner_dict_factory : function, (default: dict)
144  Factory function to be used to create the adjacency list
145  dict which holds edge data keyed by neighbor.
146  It should require no arguments and return a dict-like object
147 
148  edge_attr_dict_factory : function, (default: dict)
149  Factory function to be used to create the edge attribute
150  dict which holds attribute values keyed by attribute name.
151  It should require no arguments and return a dict-like object.
152 
153  graph_attr_dict_factory : function, (default: dict)
154  Factory function to be used to create the graph attribute
155  dict which holds attribute values keyed by attribute name.
156  It should require no arguments and return a dict-like object.
157 
158  Typically, if your extension doesn't impact the data structure all
159  methods will inherit without issue except: `to_directed/to_undirected`.
160  By default these methods create a DiGraph/Graph class and you probably
161  want them to create your extension of a DiGraph/Graph. To facilitate
162  this we define two class variables that you can set in your subclass.
163 
164  to_directed_class : callable, (default: DiGraph or MultiDiGraph)
165  Class to create a new graph structure in the `to_directed` method.
166  If `None`, a NetworkX class (DiGraph or MultiDiGraph) is used.
167 
168  to_undirected_class : callable, (default: Graph or MultiGraph)
169  Class to create a new graph structure in the `to_undirected` method.
170  If `None`, a NetworkX class (Graph or MultiGraph) is used.
171 
172  Examples
173  --------
174

176  attributes by using a single attribute dict for all edges.
177  This reduces the memory used, but you lose edge attributes.

193  creating graph subclasses by overwriting the base class `dict` with
