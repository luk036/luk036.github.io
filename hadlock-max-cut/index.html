The MAX-CUT Problem
Partition vertices into two sets (S, V \ S) to maximize the total weight of crossing edges.
MAX-CUT(G) = max ∑ w(u,v)  for  u∈S, v∉S
🍽️

Split friends across two tables to maximize cross-table conversations

Faces
A planar embedding divides the plane into faces
Each face is a cyclic sequence of boundary nodes
The outer face wraps around the sphere's back —
just another face on a globe!
Planar graphs have a dual graph structure — the key to Hadlock's polynomial-time algorithm.
Odd Faces — The Source of Frustration
⬜
Even Face
4 edges → perfectly 2-colorable
A → B → A → B
All edges cross the cut
🔺
Odd Face Frustrated
3 edges → impossible to 2-color
A → B → A → A (collision!)
At least 1 edge must be excluded
Handshaking Lemma: Every planar graph has an even number of odd faces. They always come in pairs. → Perfect matching is always possible!
Hadlock's insight: The whole problem reduces to resolving odd-face frustration.
Dual Graph Construction
Faces become vertices. Shared boundaries become edges.

Algorithm Flow
1. Check planarity — nx.check_planarity()
2. Find faces — walk half-edge boundaries
3. Build dual graph — faces → vertices
4. Identify odd faces — |boundary| mod 2 = 1
5. All-pairs shortest paths between odd faces
Shortest path between two odd faces = cheapest set of primal edges to exclude from the cut.
MWPM → Excluded Edges → MAX-CUT

The Irony
We run a complex algorithm to find the minimum total weight
so we can throw it away.
MAX-CUT = All edges \ Excluded edges
Minimize excluded → Maximize cut
The Bottleneck: Cubic Scaling
All-pairs shortest paths on the dual graph is O(V³)
😓
A circuit with 10× more faces
= 1000× longer to compute
Months, not seconds
🍫
Chocolate Bar Analogy
Snap along the grooves (articulation points)
Break into bite-sized pieces
Solve each independently
Articulation point: A single node whose removal disconnects the graph.
MAX-CUT is additive over biconnected components — no cycles span two components!

Tarjan's Algorithm
DFS-based bridge finding:
1. Track discovery time (disc)
2. Track low-link values
3. If low[v] ≥ disc[u] → articulation point
4. Pop edges from stack → one BC component
BC components are edge-disjoint — solve each independently, then union the cuts.
MWPM: Exact DP vs Greedy
BC decomposition shrinks odd-face count per block — enabling exact matching
🎯
The 18 Threshold
Most BC blocks have k ≤ 18 odd faces after decomposition
Below 18: Exact DP (2k · k²)
Above 18: Greedy (k² log k) approximation
⚡
Why This Matters
Greedy = short-sighted, loses precision
Exact DP = explores all combinations
Guaranteed optimal matching
Speed + Absolute precision
= The holy grail of optimization
BC decomposition doesn't just speed things up — it enables mathematically perfect MWPM.

When to Use BC Decomposition
✅
Good for BC
• Large graphs (1000+ nodes)
• Sparse circuit netlists
• Hierarchical designs
• Many articulation points
❌
No Benefit
• Single BC component
• Small graphs (<50 nodes)
• Dense planar graphs
• Triangulated meshes
Key: "Use the right tool for the exact shape of the problem."
BC overhead on a single-component graph just adds unnecessary cost.
