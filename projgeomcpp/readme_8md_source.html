2 
3 ![Build Status](https://travis-ci.org/luk036/projgeomcpp.svg?branch=master)
4 
5 This is an empty frame for project in C++. It should help to start a new project without caring much about project/build environment setup.
6 
7 ## Features
8 
9 * Modern, simple CMake build scripts, divided into several components for readability.
10 * Sane and simple targets for everything (see *targets* section).
11 * CPack script which can produce **deb**, **rpm**, **windows installer**, **bundle app**, **dmg** and various compressed packages.
12 * Tests via CTest.
13 * Unit tests via Catch.
14 * Documentation generation via Doxygen (code + wiki + UML diagrams with Graphviz and PlanUML).
15 * Static analysis support (clang-tidy, cppcheck).
16 * Dynamic analysis helpers via sanitizers (see cmake/sanitizers.cmake).
17 * Works with your favourite linux distro, Windows and MacOS.
18 * Gitlab CI and Travis CI configuration examples.
19 * Support for gold linker for faster linking time.
20 * Helpers for adding external dependencies downloaded during configuration or build.
21  * see `external/CMakeLists.txt` and `cmake/cleanCppExtension.cmake`

54 ## Building instructions
55 
56 ### Prerequisites
57 
58 * CMake 3.2 and newer
59 * Compiler with support for C++14
60 * git - for downloading external resources
61 * Doxygen for docs (*Graphviz for more graphs in docs, PlantUML for more UML diagrams*, PlantUML needs java)
62 * clang-tools for static analysis and formatting
63 * cppcheck for another static analysis
64 
65 #### Prerequisites on Linux
66 
67 * Arch Linux: `sudo pacman -S cmake g++ graphviz git clang clang-tools-extra cppcheck java-runtime-common cppcheck`
68  * download plantuml.jar and have it somewhere where *PATH* points to
69 
70 * Ubuntu 16.04: `sudo apt-get install cmake g++ graphviz plantuml git clang clang-tidy clang-format cppcheck`
71 
72 #### Prerequisites on Windows
73 
74 Two ways, which were tested:
75 
76 * msys2 based
77  * Minimal: `pacman -S cmake g++ git`
78  * Additional software `pacman -S clang mingw-w64-x86_64-clang-tools-extra mingw-w64-x86_64-clang-analyzer doxygen`
79  * For graphs in documentation install Graphviz (to `c:\Program Files\Graphviz`, so scripts can find it) and add its `bin` subdirectory to *PATH*, install java (have its bin directory on *PATH*), download PlantUML jar file and have it on *PATH*.
80 
81 * Microsoft Visual Studio
82  * Install cmake
83  * Install git
84  * For additional features install clang with tools, doxygen, graphviz, plantuml, java and add them to the PATH (*not tested*)
85 
86 #### Prerequisites on macOS
87 
88 * macOS Sierra `brew install llvm doxygen cppcheck graphviz`

90 ### Build on Linux & macOS
91 
92 Standard Makefiles:
93 ~~~
94 mkdir build ; cd build
95 cmake ..
96 make -j8
97 ~~~
98 
99 Ninja build witch clang, build all+doc and install it to dist folder:
100 ~~~
101 mkdir build ; cd build
102 cmake -GNinja -DCMAKE_CXX_COMPILER="clang++" ..
103 ninja all doc install
104 ~~~
105 
106 Release build which will install itself into `/usr` after `sudo make install`
107 ~~~
108 mkdir build ; cd build
109 cmake .. -DCMAKE_INSTALL_PREFIX="/usr" -DCMAKE_BUILD_TYPE="Release"
110 make -j8 all
111 sudo make install
112 ~~~

115 ### Build on windows
116 
117 #### Build on windows using MSYS2 + ninja
118 
119 On windows you should prefer Ninja since it is much faster than make (but has no color in shell).
120 
121 With gcc:
122 ~~~
123 mkdir build ; cd build
124 cmake -GNinja ..
125 ninja install
126 ~~~
127 
128 You can use `cmake -G "MSYS Makefiles" ..` if you want to use make.
129 
130 #### Build on Windows using Microsoft Visual Studio 14
131 
132 First you may wan to change project name in main `CMakeLists.txt`. Just go to the folder with the project and create new directory and create project files with this:
133 
134 ~~~
135 mkdir buildmsvc
136 cd buildmsvc
137 cmake -G "Visual Studio 14 2015" ..
138 ~~~
139 
140 and you can now open a `.sln` file with Visual Studio. You need to right click on executable target and *Set as StartUp project*. To really see the console window you want to do *Start without debugging*, since when run in debug mode only, console widow is closed too fast.

142 ### Targets
143 
144 *Note:* Availability of some targets depends on availability certain executables (e.g. clang-format for *format* target)
145 
146 * Build
147  * *all* (the default if no target is provided)
148  * *clean*
149  * *install* - install binaries into *CMAKE_INSTALL_PREFIX*
150  * *projGeomApp* - build projGeomApp binary
151  * *projGeomApp-run* - build, install and run projGeomApp binary (for your convenience)
152  * *projGeomApp-gdb* - build, install and run projGeomApp binary in gdb (for your convenience)
153  * *run* - alias for projGeomApp-run (in order to keep it short)
154 * Testing
155  * *check* - run whole test suite (see test/CMakeLists.txt)
156  * *checkVerbose* - run whole test suite (see test/CMakeLists.txt), but more verbose
157  * *unit* - build and run unit tests only (see test/CMakeLists.txt)
158  * *unitall* - same as previous, only prints even successful unit tests results
159 * Releases
160  * *package* - builds packages (zip,deb,rpm,windows installer depending on OS)
161 * Miscellaneous
162  * *doc* - build documentation (if doxygen is available)
163  * docs can be found in `build_dir/doc/doc/index.html`
164  * if you want them to be a part of install package, uncomment section at the end of `doc/CMakeLists.txt`
165  * *format* - run clang-format on all source files (.clang-format in root directory of a project is used)
166  * *gdb* - run target executable with gdb (does not work from ninja, needs gdb)
167 * Static analysis
168  * *tidy* - run clang static analysis on all sources
169  * *cppcheck* - call cppcheck on all files (another static analysis)
170 * External
171  * *update* - update all external sources/projects

174 ### CMAKE variables
175 
176 * `-DCMAKE_INSTALL_PREFIX` - location for installation
177 * `-DVERSION_HOST` - build machine name, see version.h.in
178 * `-DCMAKE_BUILD_TYPE` - for build type
179 * `-DMEASURE_ALL=ON/OFF` - Measure time of all compilations, best used with single threaded build. Needs `time` command (unix).
180 * `-DUSE_GOLD_LINKER=ON/OFF` - whether to link with Gold linker.
